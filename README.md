# Distributed Audio System

分布式音频系统 - 通过 WiFi 将电脑音频分发到多个安卓设备播放

## 项目简介

本项目实现了一个基于 WiFi 局域网的分布式音频系统，可以将 Windows 电脑的系统音频实时传输到多个安卓设备进行播放，支持立体声、2.1、5.1、7.1 等多声道配置。

## 技术架构

### Windows 发送端 (WindowsSound/)

- **WASAPI Loopback** - 捕获系统音频输出
- **Opus 编码** - 低延迟音频压缩 (20-50ms)
- **mDNS/Bonjour** - 局域网设备发现
- **UDP/RTP** - 实时音频传输
- **WPF UI** - 现代化用户界面

### 安卓声音端 (AndroidSoundPlayer/)

- **自定义协议接收** - 超低延迟音频接收
- **DLNA 支持** - 兼容标准 DLNA 设备
- **低延迟播放器** - 基于 AudioTrack 优化
- **自适应缓冲** - 抖动缓冲平滑网络波动
- **固定设备标识** - UUID + 可配置别名

### 安卓配置端 (AndroidController/)

- **设备扫描** - 自动发现局域网声音端
- **工作模式配置** - 声卡模式 / DLNA 模式
- **多机配置** - 左右声道分配
- **延迟校准** - 多设备同步调整

## 技术特性

| 特性 | 说明 |
|------|------|
| 音频延迟 | 20-50ms (自定义协议) / 200-500ms (DLNA) |
| 音频质量 | Opus 编码，支持高码率 |
| 多声道 | 立体声 / 2.1 / 5.1 / 7.1 |
| 网络协议 | UDP + 自定义RTP / DLNA |
| 设备发现 | mDNS/Bonjour |
| 同步精度 | <5ms (延迟补偿算法) |

## 系统要求

- **Windows**: Windows 10/11, .NET 8.0
- **安卓声音端**: Android 4.4+ (API 19+)
- **安卓配置端**: Android 5.0+ (API 21+)
- **网络**: WiFi 局域网，建议 5GHz 频段
