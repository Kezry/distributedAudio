# Distributed Audio System - Integration Testing Guide

## 测试环境配置

### 硬件要求
- Windows 10/11 PC x 1 (发送端)
- Android 设备 x 2-3 (声音端)
- Android 设备 x 1 (配置端)
- WiFi 路由器 (5GHz 推荐)

### 软件版本
- WindowsSound: 1.0.0
- AndroidSoundPlayer: 1.0.0
- AndroidController: 1.0.0

---

## 1. 端到端延迟测试

### 测试目标
- 端到端延迟 < 50ms
- 同步精度 < 5ms

### 测试步骤
1. 启动 WindowsSound 发送端
2. 启动 2 个 AndroidSoundPlayer 设备
3. 播放测试音频
4. 使用示波器或专用设备测量延迟

### 测试工具
- **音频分析软件:** Audacity
- **延迟测量:** RTL-SDR + 分析工具
- **网络分析:** Wireshark

### 预期结果
```
配置: 立体声，48kHz，Opus 64kbps
预期延迟: 20-40ms
同步精度: < 5ms
```

---

## 2. 同步精度验证

### 测试方法
1. 配置两个设备为左/右声道
2. 播放包含脉冲信号的测试音频
3. 录制两个设备输出
4. 分析两个声道的时间差

### 验证标准
- **左右声道偏差:** < 5ms
- **相位一致性:** > 95%
- **长时间稳定性:** 1小时内偏差变化 < 10ms

### 测试音频
- 1kHz 正弦波
- 白噪声脉冲
- 音乐片段 (流行/古典)

---

## 3. 网络抖动测试

### 测试场景
- 正常网络条件
- 拥堵网络 (模拟)
- WiFi 信号弱化

### 测试工具
- **网络模拟:** Clumsy (Windows)
- **QoS 设置:** 路由器配置
- **监控:** Wireshark

### 测试参数
```
延迟波动: 0-50ms (随机)
丢包率: 0%, 1%, 5%
带宽限制: 10Mbps, 20Mbps, 50Mbps
```

### 验证标准
- **抖动缓冲:** 自适应调整
- **音频断续:** < 1% 时间
- **恢复时间:** < 1秒

---

## 4. 音频质量评估

### 评估方法
- **主观测试:** A/B 对比
- **客观测试:** PESQ, POLQA

### 测试音频
| 类型 | 采样 |
|------|------|
| 人声 | 播客/有声书 |
| 音乐 | 流行/摇滚/古典 |
| 电影 | 动作片对白 |

### 评估指标
- **清晰度:** 语音清晰
- **保真度:** 还原原始音频
- **立体声:** 声道分离度
- **动态范围:** 低频/高频表现

### MOS 评分标准
| 分数 | 质量 |
|------|------|
| 5.0 | 优秀 |
| 4.0+ | 良好 |
| 3.5+ | 可接受 |
| < 3.5 | 需改进 |

## 5. 性能优化测试

### Windows 端优化
- CPU 使用率 < 20%
- 内存使用 < 200MB
- 延迟稳定 (标准差 < 5ms)

### Android 端优化
- CPU 使用率 < 30%
- 内存使用 < 150MB
- 电池消耗优化
- **目标:** 4 小时连续播放 < 30% 电量

### 优化策略
1. **编码参数调整**
   - 比特率自适应
   - 帧大小优化

2. **网络优化**
   - UDP 缓冲区调优
   - 丢包恢复策略

3. **播放优化**
   - AudioTrack 参数配置
   - 线程优先级设置
